Commit Archaeologist

A command-line tool for git history

Every commit leaves a layer.

Point it at any repository and it excavates the record: who wrote what, which files keep getting rewritten, when the work actually happens, and which commit messages didn't leave much for future you to go on.

Get started See what it reports
$ pip install commit-archaeologist
a repository read as sediment — recent work at the surface, the first commit at bedrock

What gets uncovered

five layers, one dig
SURFACE

Commit rhythm

Which weekdays and hours the work actually happens, so the pattern of a project's activity is visible at a glance — not just its total volume.

Monday ████████████ 24 Tuesday ██████████████████ 31 Wednesday ██████ 11
LAYER II

Contributor strata

Commits, insertions, deletions, and files touched, broken down per author — who carried a given stretch of the project's history.

rynix27 142 commits +8,410 / -3,220 a.dev 37 commits +1,105 / -640
LAYER III

Hotspot deposits

Files that get rewritten again and again. Usually core logic, a shared utility, or a spot that keeps causing trouble.

src/analyzer.py 38 commits +612/-201 src/report.py 21 commits +340/-98
LAYER IV

Message fossils

Commit messages preserved exactly as they were written — including the ones that didn't leave much for anyone to go on later.

a1b2c3d "fix" 2025-11-02 9f8e7d6 "wip" 2025-11-04
BEDROCK

Origin point

The date range and total commit count anchor everything else — the span of time a repository's story actually covers.

214 commits 2024-02-11 → 2026-06-30

Usage

runs against any local repository
commit-archaeologist
analyzes the current directory
commit-archaeologist ./my-project --since 2025-01-01
limits the dig to commits after a given date
commit-archaeologist ./my-project --top 20 --export report.md
shows the top 20 entries per list and writes a Markdown report to disk

Install

Python 3.9+
$ pip install commit-archaeologist
